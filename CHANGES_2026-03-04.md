# Changes — 4 March 2026

Summary of work completed on this date: curriculum UI improvements, instructor dashboard and learners page made dynamic, API path fix, and course editor Analytics tab made dynamic.

---

## 1. Instructor Dashboard Overview — Dynamic

- **Backend:** New `GET /api/instructor/stats` (controller: `server/controllers/instructor-controller/stats.controller.js`, route: `server/routes/instructor-routes/stats.routes.js`).
- Returns for the logged-in instructor: total learners, total/published/draft/pending/rejected course counts, total revenue, average rating (from CourseReview), top 5 courses by learners, recent 5 enrollments.
- **Frontend:** `InstructorDashboard.jsx` fetches from `/instructor/stats`, shows real stat cards, course breakdown strip, top performing courses table (with thumbnail, status, learners, revenue), quick actions (New Course, View Learners), and recent enrollments feed. Loading skeletons and error state added.

## 2. Instructor Learners Page — Dynamic

- **Backend:** New `GET /api/instructor/learners` (controller: `server/controllers/instructor-controller/learners.controller.js`, route: `server/routes/instructor-routes/learners.routes.js`).
- Returns all learners enrolled in the instructor’s courses, with user info (name, email), course count, average progress, per-course progress, last active, enrolled since.
- **Frontend:** `LearnersList.jsx` fetches from `/instructor/learners`, shows search (name/email), sortable columns (Learner, Courses, Avg. Progress, Last Active), expandable rows with per-course progress and thumbnails, email (mailto) action, loading/error/empty states.

## 3. API Path Fix — Resource Not Found (404)

- **Issue:** Instructor dashboard and learners pages returned “Resource not found” because the client was calling paths that doubled the `/api` segment.
- **Fix:** `apiClient` uses `baseURL: 'http://localhost:5000/api'`, so request paths must not include `/api`. Updated:
  - `LearnersList.jsx`: `/api/instructor/learners` → `/instructor/learners`
  - `InstructorDashboard.jsx`: `/api/instructor/stats` → `/instructor/stats`

## 4. Course Editor Analytics Tab — Dynamic

- **Backend:** New `GET /api/courses/:id/analytics` (controller: `server/controllers/instructor-controller/courseAnalytics.controller.js`), protected with `requireCourseOwner('id')`.
- Returns for the course: completion distribution (0–1%, 1–25%, 26–50%, 51–75%, 76–99%, 100%), average completion rate, chapter completion (per chapter: completed count, total learners, %), leaderboard (rank, name, email, completion %, purchased date, last progress date).
- **Frontend:** `ReportTab.jsx` receives `courseId` from `EditorTabPanels`, fetches `/courses/${courseId}/analytics`, and renders:
  - **Course Completion:** Percentage breakdown table and doughnut chart for average completion rate and total learners.
  - **Chapter Completion:** Table of chapters with “Users completed” and “%”; empty state when no chapters/learners.
  - **Leaderboard:** Sortable table, search by name/email, Export CSV (filtered list). Loading and error states; “Save your course to view analytics” when `courseId` is missing.
